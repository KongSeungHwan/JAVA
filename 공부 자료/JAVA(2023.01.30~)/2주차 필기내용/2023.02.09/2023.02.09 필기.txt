추상 클래스

추상메소드가 포함되어있는 클래스를 추상 클래스라고 한다.

또한 추상메소드란? 중괄호 부분이 없는 미구현된 메소드를 말한다.

추상클래스는 왜 쓰이는가?

같은 종류의 객체지만 기능을 달리해야하는 부분이 있을 경우 사용

오로지 상속만을 위해 존재하는 클래스!(객체 생성 불가)

EX) COMPUTER >> DESKTOP,NOTEBOOK 
데스크탑이랑 노트북은 전원 키는 방법이 조금씩 다르다!
데스크탑은 본체 전원 버튼을 눌러야지만 작동하는 형태고
노트북은 열기만하면 전원이 켜지는 형태를 구현하고 싶을때
추상클래스를 사용.
그러한 차이를 보여주기 위해 추상 클래스를 사용하는 것이다!

***다형성 활용:
배열에 객체를 저장 시에 만일 메소드를 사용하지 않고, 저장하고 정보조회 용도로만 사용할 객체면
자료형을 Object로 사용하여도 된다.(다형성 응용)







인터페이스 구성요소

인터페이스 안에 메소드를 구현하지 않고 만든다면 무조건
 public이나 abstract를 추가 해주는 기능을 가지고 있다.(무조건 public이며 추상 메소드


상수 
추상메소드
디폴트 메소드
PRIVATE 메소드
PRIVATE STATIC 메소드 

이중에서 PRIVATE ,PRIVATE STATIC으로 된 메소드는 디폴트 메소드 안에서만 사용가능하다!

인터페이스와 추상 클래스의 차이?
둘 다 유사하나 예약어 차이나 (EXTENDS,IMPLEMENTS ), 추상클래스에서 상속받은 클래스는 상속받은 메소드와
해당클래스에서 선언한 메소드를 섞어 사용가능하나, 
인터페이스같은 경우는 상속 받은 클래스가 인터페이스 내에서 명시되어있는 메소드만을 오버라이딩해서 사용할 수 있다는 차이점이 있다.
(다른 메소드 사용불가)

변수로 static 형으로 하는 경우 static 메모리에 미리 할당이 되어 모든 영역에서 공용으로 사용가능하나
클래스를 static 형으로 하는 경우는 다르다.

static형의 클래스면.. 모든 영역에서 해당 자료형으로 해당 객체를 생성하여 필드나 메소드에
접근한다.
다른 곳에서도 공유가 되긴 하나 취급은 다른 클래스의 객체 취급으로 해야한다.




Object 클래스의 메소드를 사용할때 각자 다른 자료형을 가지고 있으므로 각자 자료형에 맞게 오버라이딩을 시켜 사용해주는 게 좋다. 
>>Object클래스의 equals 함수,toString 함수 오버라이딩 해봄.
