JSP를 알기전에!


클라이언트: 데이터를 요청하는 컴퓨터 측을 의미
서버: 클라이언트 컴퓨터에서 요청 값을 받아와서 데이터를 처리하는 컴퓨터 측을 의미
언제든지 서버, 클라이언트 측이 바뀔 수 있다.

ex)
회원 가입 절차
컴퓨터1       컴퓨터2
1. 회원 가입을 위해 컴퓨터 1이 회원가입을 하겠다 요청.(컴퓨터 1이 클라이언트, 컴퓨터2 서버)

2.컴퓨터 2가 회원가입을 위해 개인정보가 필요하다 요청(컴퓨터 2가 클라이언트, 컴퓨터 1 서버)


클라이언트 컴퓨터가 서버 컴퓨터에 데이터를 요청할 때 요청/응답방식을 규정해놓은 규칙이 있다.

그것이 HTTP(Hyper Text Tranfer Protocol)라고 함.

요청방식
GET, POST


GET
서버의 데이터를 검색할 때 사용하는 방식이다.
url로 요청시에 입력정보가 보인다.

POST
서버의 데이터를 생성/수정할때 사용하는 방식이다.

JSP 작동원리

웹브라우저 요청
1.http://~~~/~~.jsp 해당url을 서버 컴퓨터에 요청한다.(request)

2.컨테이너가 jsp >> sevlet 으로 바꿔준다.

3.컴파일러가 servlet >> class 파일로 변환.

4.서버에서 웹브라우저를 통해 html로 응답 (reponse)

ex)
url 주소창에 naver.com(request)으로 치면 네이버의 서버 컴퓨터에서 
html로 네이버 첫페이지를 보여주는 것(response)

그래서 JSP란?(Java Server Page)
HTML 코드에 Java 코드를 넣어 동적인 웹 페이지를 생성하는 웹 어플리케이션 도구



URL(Uniform Resource Locator)은 인터넷에서 웹 페이지, 이미지, 비디오 등 리소스의 위치를 가리키는 문자열

프론트엔드 vs 백엔드

프론트 엔드는 클라이언트에게 보여지는 디자인같은 것들을 다루는 분야

백엔드는 클라이언트에게 안보이는 데이터를 처리하는 분야

두 분야를 모두 가지고 있으면 풀 스택 

요청방식(*get,post)



HTML 파일 : 확장자가 "html"인 파일로서 클라이언트의 브라우저에 의해서 내용을 해석하여 실행하며 
자바나 톰캣 서버가 설치되어 있지 않아도 정상적으로 실행된다.
JSP 파일 : 톰캣 서버가 번역하여 그 결과를 HTML 태그로 변환한 후 웹브라우저에 내려 보내는데, 
톰캣이 동작시키기 위해서는 JSP 파일은 확장자가 "jsp"이어야 하고 페이지에 JSP인 것을 알리는 페이지 지시자인
 <%@ page%>가 반드시 있어야 한다.
charset :웹 브라우저 혹은 HTML문서가 어떠한 문자셋으로 저장되었는지 명시해놓은 속성.
유니코드 인코딩: 사용 중인 거의 모든 세계 언어의 문자를 포함하는 고정 길이의 문자 인코딩 체계 

호스팅(Hosting): 서버 컴퓨터의 전체 또는 일정 공간을 이용할 수 있도록 임대해 주는 서비스

EUC-KR: Extend unix code의 약자로 (영어를 제외)한 문자를 표현하기 위한 확장부호를 의미

UTF-8: 가장 많이 사용되는 가변 길이 유니코드 인코딩
컨테이너: 어떤 환경에서나 실행하기 위해 필요한 모든 요소를 포함하는 소프트웨어 패키지

캐시: 자주 사용하는 데이터나 값을 복사해 놓는 임시 저장소
(컴퓨터의 데이터 처리 속도를 극대화 하기위한 임시 저장공간)

버퍼: 데이터를 한 곳에서 다른 곳으로 전송하는 동안 일시적으로 그 데이터를 보관하는 메모리 영역
(처리속도 차이를 줄이기 위한 임시 저장 공간)

JSP VS HTML
JSP : 서버 측에서 동작해야 할 코드들이 있을 경우 사용한다.
 서버에서 보낸 데이터에 따라 값이 바뀔 수 있는 변수에 저장된 내용들을 출력할 때 사용한다.
HTML : 변화가 없는 단순 상수값을 출력할 때 사용한다.

클라이언트 컴퓨터 자원으로 충분히 표현이 되는 경우 HTML 사용
서버 컴퓨터 자원이 필요할 경우 JSP 사용


내부객체
개발자가 객체를 생성하지 않고 바로 사용할 수 있는 객체

쿼리 스트링 (Query String):사용자가 입력 데이터를 전달하는 방법 중의 하나로, 
url 주소에 미리 협의된 데이터를 파라미터를 통해 넘기는 것을 말한다.

WAS(Web Application Service): 동적인 데이터를 처리해주는 서버

MIME 타입: 클라이언트에게 전송된 문서의 다양성을 알려주기 위한 메커니즘
